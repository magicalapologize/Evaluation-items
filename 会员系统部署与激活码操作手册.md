# 云渡超级会员系统部署与激活码操作手册

## 一、系统规则

- 会员只使用激活码注册和登录，不需要手机号、用户名或密码。
- 首次输入未使用的激活码时，系统自动创建会员并开始计算有效期。
- 以后继续输入首次激活时的同一枚激活码，即可在新设备登录。
- 登录成功后浏览器保存安全会话 Cookie，有效期 30 天。
- 同一会员最多保留 2 个有效设备会话，第三次登录会退出最早的设备。
- 会员可以无限次使用当前和后续上线的全部测试。
- 游客仍然可以使用每款产品对应的每日测试码。

## 二、会员档位

| 类型参数 | 展示名称 | 有效期 | 当前价格 |
|---|---|---:|---:|
| `monthly` | 月度会员 | 30 天 | 12.9 元 |
| `quarterly` | 季度会员 | 90 天 | 29.9 元 |
| `annual` | 年度会员 | 365 天 | 79.9 元 |
| `lifetime` | 终身会员 | 网站持续运营期间 | 199 元 |

所有档位登录后统一展示为“云渡超级会员”。

## 三、首次上线部署

### 1. 检查当前项目

```bash
cd "05.我的产品/虚拟产品/产品制作/网站汇总/Evaluation-items"
npx wrangler deploy --dry-run
```

确认输出中存在：

```text
env.DB (yundu-evaluation)
env.ASSETS
```

### 2. 给线上 D1 创建会员表

先执行迁移，再部署代码：

```bash
npx wrangler d1 migrations apply yundu-evaluation --remote
```

看到 `0002_member_system.sql` 状态为成功即可。

### 3. 部署 Worker 和静态页面

```bash
npx wrangler deploy
```

部署后依次访问：

```text
https://magicassess.top/
https://magicassess.top/member/
https://magicassess.top/api/member/me
```

未登录访问 `/api/member/me` 应返回：

```json
{"success":true,"member":{"authenticated":false,"active":false}}
```

## 四、每天生成并导入会员激活码

会员激活码不是每日通用码。每一笔会员订单必须发放一枚不同的激活码。

### 1. 生成激活码

生成 10 个月度会员码：

```bash
node scripts/generate-member-codes.mjs --plan monthly --count 10
```

生成 10 个季度会员码：

```bash
node scripts/generate-member-codes.mjs --plan quarterly --count 10
```

生成 20 个年度会员码：

```bash
node scripts/generate-member-codes.mjs --plan annual --count 20
```

生成 5 个终身会员码：

```bash
node scripts/generate-member-codes.mjs --plan lifetime --count 5
```

脚本会在网站项目外部创建：

```text
Evaluation-items-private/member-codes/
```

每次生成两份文件：

- `.csv`：原始激活码发货清单，必须保密。
- `.sql`：只有激活码摘要，可以安全导入 D1。

### 2. 导入线上 D1

复制脚本输出的 `.sql` 完整路径：

```bash
npx wrangler d1 execute yundu-evaluation --remote --file "生成的SQL完整路径"
```

不要把 `.csv` 文件导入数据库，也不要放进网站目录、GitHub 或公开网盘。

### 3. 检查库存

```bash
npx wrangler d1 execute yundu-evaluation --remote --command "SELECT plan_type, status, COUNT(*) AS count FROM member_activation_codes GROUP BY plan_type, status ORDER BY plan_type, status;"
```

重点关注 `status = unused` 的数量。

## 五、小红书发货流程

1. 买家购买对应会员档位。
2. 打开私密 `.csv` 发货清单。
3. 选择一枚状态为 `unused` 的激活码。
4. 将激活码和会员中心地址发给买家。
5. 在自己的发货清单中标记订单号和已发货，避免重复发送。

建议发货内容：

```text
云渡超级会员激活码：会员唯一激活码

激活地址：
https://magicassess.top/member/

首次输入会自动开通会员，以后仍使用同一枚激活码登录，请妥善保存，不要转发给他人。
```

## 六、会员续费规则

- 会员未过期：新时长从原到期时间后继续累加。
- 会员已过期：新时长从续费当天重新计算。
- 输入终身会员码：升级为终身会员。
- 终身会员页面不再允许续费。
- 用于续费的新激活码使用后不能再次使用。

## 七、退款或违规共享处理

通过激活码尾号查找会员：

```bash
npx wrangler d1 execute yundu-evaluation --remote --command "SELECT id, code_hint, plan_type, status, activated_at, expires_at FROM members WHERE code_hint = '激活码最后4位';"
```

确认会员 ID 后停用会员：

```bash
npx wrangler d1 execute yundu-evaluation --remote --command "UPDATE members SET status = 'disabled', updated_at = datetime('now') WHERE id = '会员ID'; UPDATE member_sessions SET revoked_at = datetime('now') WHERE member_id = '会员ID' AND revoked_at IS NULL;"
```

恢复会员：

```bash
npx wrangler d1 execute yundu-evaluation --remote --command "UPDATE members SET status = 'active', updated_at = datetime('now') WHERE id = '会员ID';"
```

## 八、日常检查

查看会员数量：

```bash
npx wrangler d1 execute yundu-evaluation --remote --command "SELECT plan_type, status, COUNT(*) AS count FROM members GROUP BY plan_type, status;"
```

查看最近激活记录：

```bash
npx wrangler d1 execute yundu-evaluation --remote --command "SELECT code_hint, plan_type, activated_at, expires_at FROM members ORDER BY activated_at DESC LIMIT 20;"
```

清理 30 天以前的登录尝试记录：

```bash
npx wrangler d1 execute yundu-evaluation --remote --command "DELETE FROM member_auth_attempts WHERE attempted_at < datetime('now', '-30 days');"
```

清理已经过期或被撤销 30 天以上的会话：

```bash
npx wrangler d1 execute yundu-evaluation --remote --command "DELETE FROM member_sessions WHERE expires_at < datetime('now', '-30 days') OR revoked_at < datetime('now', '-30 days');"
```
